ZuiperfCtl 按钮说明 - v5

一、顶部

静音通知
- 用来刷新或重新拉起 ZuiperfCtl 的常驻低打扰通知。
- 通知本身无声音、无震动，属于安卓的静音通知。
- 通知栏里可以直接点 60、90、120、144、默认，快速切换全局刷新率。

二、左侧应用列表

用户
- 只显示普通用户安装的应用。
- 游戏一般优先在这里找。

系统
- 显示系统应用。
- 如果要给官方游戏助手、便签、系统服务等做测试，可以在这里找。

已配置
- 只显示已经保存过 ZuiperfCtl 档案的应用。
- 后续排查自己给哪些应用设置过规则，优先看这里。

全部
- 用户应用和系统应用都显示。

搜索应用或包名
- 输入应用名称或包名来过滤列表。
- 包名更稳定，例如 com.kurogame.mingchao。

列表左侧勾选框
- 勾选: 给这个应用保存一条档案。
- 取消勾选: 移除这个应用的档案。
- 更精细的设置建议先点应用行，在右侧选刷新率和开关，再点“保存档案”。

三、右侧档案开关

刷新率规则
- 打开后，这个应用会写入刷新率规则。
- 开启自动切换后，前台切到该应用时，守护服务会按它的档案切刷新率。

XML 调度档案
- 预留给 ZuiPP XML 调度档案。
- 当前 v5 已经验证 XML 可以运行时 bind mount 到 /system/etc。
- 后续 XML 生成器做好后，这个开关会用于指定该应用是否参与 XML 调度。

AsoulOpt 线程档案
- 预留给 AsoulOpt 配置档案。
- 当前 AsoulOpt 已系统内置运行，v5 修复了重启方式。
- 后续如果做不同应用的线程/配置切换，会用到这个开关。

四、刷新率选择

60Hz / 90Hz / 120Hz / 144Hz
- 选择该应用档案要使用的刷新率。
- 选中后还不会立即保存，必须点“保存档案”。
- 如果点“立即锁定”，则会马上把系统刷新率锁到当前选中的数值。

五、操作按钮

保存档案
- 把当前选中应用的刷新率和三个开关保存到守护服务。
- 如果只开“刷新率规则”，会同时更新自动切换规则。

移除档案
- 删除当前选中应用的 ZuiperfCtl 档案。
- 同时删除它的刷新率自动切换规则。

开启自动
- 开启前台应用自动切刷新率。
- 开启后，守护服务会定期识别当前前台应用，匹配已保存规则。

关闭自动
- 关闭前台应用自动切刷新率。
- 关闭时会恢复系统默认刷新率设置。

立即锁定
- 不等前台应用匹配，直接把系统刷新率锁到当前选中的 60/90/120/144。
- 适合临时测试。

恢复默认
- 删除 min_refresh_rate 和 peak_refresh_rate，让系统回到默认自适应策略。

刷新状态
- 让 App 重新读取守护服务状态、当前刷新率、自动切换状态、应用档案。
- 如果你刚点过按钮但页面没更新，可以点它。

重启 AsoulOpt
- 重启系统内置的 AsoulOpt 服务。
- v5 起这个动作由 init 执行，不再由 shell daemon 直接 ctl.restart。

应用 XML
- 把 /data/local/tmp/zui_perfctl/zuipp/ 下的工作 XML bind mount 到:
  /system/etc/game_policy.xml
  /system/etc/performanceconfig.xml
- 然后重启 ZuiPP/游戏服务，让 XML 调度读取新内容。
- 不需要重刷包。

恢复 XML
- 取消上面的 XML bind mount。
- 系统会重新看到 /system/etc 里原始的 XML。
- 然后重启 ZuiPP/游戏服务。
